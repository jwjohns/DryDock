Drydock

Git-Native Deployments for Secure, Scalable Infrastructure across GCP and Azure.

Drydock is an open source deployment framework designed to separate static infrastructure ("the hull") from dynamic configuration and secrets ("the cargo"). It enables secure, scalable, and reproducible deployments using GitHub Actions, Terraform, Helm, and Workload Identity Federation (WIF), especially for teams moving away from legacy Jenkins-based deploys.

Why Drydock? Key Features & Philosophy

Workload Identity Federation

Eliminates long-lived cloud secrets in CI/CD by using OIDC tokens for secure, keyless authentication to GCP and Azure.

Git-Native Approach

Manage your infrastructure and deployments as code, with versioned, auditable changes tracked in Git.

Hull & Cargo Model

Decouples static infrastructure templates ("the Hull") from dynamic, environment-specific configurations ("the Cargo").

Multi-Cloud Ready

Native support for GCP and Azure using industry-standard tools like Terraform and Helm.

Developer Empowerment

CI/CD configuration lives in your repository, supported by local development aids for easier testing and iteration.

Secure by Design

Built with modern security best practices at its core, reducing attack surfaces and improving compliance.

Core Philosophy: In a shipyard drydock, the hull is constructed before it's launched. Drydock adopts this: The "Hull" is your static, versioned infrastructure, while "Cargo" is the runtime values and secrets injected at deployment.
